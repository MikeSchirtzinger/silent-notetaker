01 / 06Transparency

One HTML file. No build, nothing to hide.

The whole app — markup, styles, logic, even the Web Workers — lives in one file. Trivial to share, trivial to audit ("show me where the audio goes"), and impossible to slip in a phone-home. Workers are inlined as text and spun up from a Blob URL.

02 / 06Resource isolation

WebGPU for speech. WASM for everything else.

The GPU is the scarce resource, so the heaviest model — Voxtral — gets it to itself. Speaker-ID (TitaNet) and the question LLM (Qwen) run on WASM/CPU so they can never contend with the streaming loop for GPU memory or scheduling. Different jobs, different silicon.

03 / 06Bounded memory

The invisible runaway. KV cache that the profiler can't see.

Inside one generate() the KV cache grows every token — and it lives in GPU/native memory, invisible to the JS heap profiler. At ~0.52 MB/token the old 4096-token cap peaked near 2 GB → frozen tab around 5 min. Fix: two caps + context recycling, re-anchored at "now" so no audio is dropped at the seam.

04 / 06Throughput

Multithreaded WASM via cross-origin isolation.

The WASM models only go multithreaded (~3–4× faster) when the page is cross-origin isolated — which unlocks SharedArrayBuffer. That needs two response headers a file:// URL can't set. credentialless keeps the cross-origin CDN + Hugging Face model imports working.

05 / 06From scratch

Speaker ID with no off-the-shelf pipeline.

No browser library does diarization, so it's hand-built. TitaNet won a 6-model bake-off (0% EER on LibriSpeech test-clean); its mel front-end was reimplemented in JS and byte-validated against Python — cosine 1.000000. Speakers are then tracked by online leader clustering on a tuned cosine threshold.

06 / 06Local-first

The only way out is one you run yourself.

Meetings persist in IndexedDB. The optional Claude bridge is the single off-device path — off by default, talking to a server you run. It drives the claude CLI with ANTHROPIC_API_KEY scrubbed, so it uses your subscription's OAuth — never a billed key, never one stored in the repo.
